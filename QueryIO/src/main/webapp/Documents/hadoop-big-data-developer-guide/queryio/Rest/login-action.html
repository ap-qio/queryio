Authentication: Login Action

Description

QueryIO Server provides access to authorized user only. To login into QueryIO server, you have to provide username and password along with the GET request to the server.
QueryIO checks if the username & password provided are correct or not. If provided credentials are found correct then it authenticates the user by returning a authorization token which should be further used along with bucket operations and object operations for authorization.

Requests

Syntax

  • GET /queryio/ HTTP/1.1
Host: QueryIO.com
username: <username> password: <password>
  • <username>: Username of the user for login
  • <password>: Password for the username provided.

Examples

Sample Request

The following request specifies request for authentication.

  • GET /queryio/ HTTP/1.1
Host: QueryIO.com
username: admin password: admin

Sample Response

If the username and password provided in the request are correct, then QueryIO server will return the authorization token.

  • Token: iffo6l9hel2hfmbj2384joljgh9mqga58gb9if9593ucli9ke5s2e3854shhcmmm
