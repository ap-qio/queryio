Add Hive Data Definition

Description

Add Hive DataDefinition operation is used to add given hive datadefinition details to the database

Requests

Syntax

  • POST /queryio/hive/ HTTP/1.1
Host: QueryIO.com
authorization: <Token>
  • authorization: Token for authorization.

Request Headers

The POST hive definition operation in Os3/QueryIO Server uses following request header:

Hive JSON

Key Value Type Required
hiveId Unique identifier for the Hive ID.
Example: "adhocId": "Hive1" 		String Yes
namenode ID of the NameNode on which data will be queried.
Example: "nameNodeId": "NameNode1", 		String Yes
resourceManager ID of the ResourceManager on which data will be queried.
Example: "nameNodeId": "NameNode1", 		String Yes
source Path HDFS path where files exists String Yes
adHocTableName Name of the table which gets created in database with fields you pass in JSON format. String Yes
fileName Name of the sample file String Yes
filePathPattern Pattern of the sample file (Extension of file) String Yes
fields Table columns details JSONObject Yes
encoding Type of encoding.
For example : UTF-8 		String Yes
First Row Header Does Sample file first row is header or not. Boolean Yes
delimiter Sequence of one or more characters used to specify the boundary between separate, independent regions in plain text or other data streams String Yes
Value Separator (Optional)Sequence of one or more characters used to specify the boundary between separate, independent regions in plain text or other data streams String Yes
