What is QueryIO

QueryIO is a solution for Big Data Analytics. QueryIO is used to store and process vast amounts of structured and unstructured data. QueryIO is especially well suited to enable users to process unstructured data, give it a structure and support analysis of this data using standard SQL. QueryIO enables you to leverage the vast and mature infrastructure built around SQL and relational databases and utilize it for your Big Data Analytics needs.

QueryIO builds on Hadoop's scalability and reliability. QueryIO makes it easy to scale to hundreds of Compute+Store cluster nodes and peta bytes of data in an easy and logical manner. QueryIO enhances basic Hadoop to provide a comprehensive Big Data Analytics solution. Some key enhancements QueryIO provides to basic Hadoop:

  • Data Tagging and MetaData extension
  • Standard SQL interface for hadoop big data querying and analysis
  • Web-based cluster management
  • Comprehensive monitoring, including rules, alerts, etc.
  • Value added services such as S3-compatible REST API, FTP interface, etc.

    5. Optimized UI to for faster navigation and content search.
    6. Added support for Pagination across different views wherever applicable.
    7. Optimized MapReduce jobs for faster data processing.
    8. Provided additional option to process data at record level for data tagging.
    9. Optimized processing engine to have single MR job create multiple data tags in parallel.
    10. Added support to start/stop cluster with single click.
    11. Provided support to create a new hive DDL through QueryIO REST apis.
    12. Fixed most known bugs.

    Major changes from QueryIO v 2.0 to 2.5:

    1. Data Tagging feature which provides users an extensive Data Intelligence framework enabling users to create data tags in MetaStore DB for files on HDFS cluster. Both On-Ingest and Post-Ingest processing is supported.
    2. Enhanced Hive to support recursive processing of folders / files.
    3. Enhanced Hive to understand file path filter patterns, restricting the processing of files to just user defined file path pattern.
    4. Support to extract FilePath as well during Hive Queries.
    5. Support for Input Path Filter while executing Hive Queries. This has an advantage where you can use Hive to process just subset of files based on query results from MetaStore.
    6. Support for MySQL database along with PostgreSQL out of the box.
    7. Implemented Generic framework to enable user to configure other databases to use with QueryIO.
    8. Separation of result set from Report Viewer, enabling analysis of same query results with Spreadsheet and other report formats like html, pdf and xls.
    9. Support for cluster setup during installation and fixing other know issues with the installer.
    10. Support for local cluster installation for evaluation users. Local cluster installation now supports 127.0.0.1/localhost which allows the cluster to work fine even when the IP address of the installation machine changes. This makes evaluation process easier for users connecting to networks allocating dynamic IP addresses, or for users connecting to different networks at home and office.
    11. Support for upgrading from one version of QueryIO to another. This enables user to persist all his data on cluster / user reports / queries / cluster configurations etc. created in older installation.
    12. Fixed all known issues with uninstall process where QueryIO takes care of uninstalling all the services cleanly along with Remote cluster uninstallation. It also now supports persisting user's cluster data and reports.
    13. Fixed most known bugs.

    1. Upgraded to Hadoop 2.0.3-alpha.
    2. Added support for automated Hadoop cluster setup (start NameNode, DataNode, ResourceManger, etc. services) on single and multiple hosts.

    4. Migrated from HSql to PostgreSQL 9.2.3 for better performance, security and scalability.
    5. Added support for data import/export to/from additional data sources such as HDFS, POP / IMAP, HTTP/HTTPS, SSH, SFTP, Amazon S3, etc.

    9. Improved charting capabilities by adding 9 new chart types (area, gantt, scatter, etc.) and enabling additional operators/functions to be used for charting.
    10. Added support to print / email reports.
    11. Added support for configuration of Journal Nodes for HDFS NameNode High Availability.
    12. Added Disaster Recovery module to backup and restore NameNode's FSImage using QueryIO MetaStore.
    13. Added automated diagnosis and repair feature to keep QueryIO MetaStore in sync with NameNode's FSImage.
    14. Enhanced data management usability: Added support to save / configure data source connections used during import / export, restructure UI, etc.
    15. Completely restructured the menus for better user experience.
    16. Fixed most known bugs.
